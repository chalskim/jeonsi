# 전시(專時)용으로 새로 SRS 버전의 “플랫폼 개발 요구사항 명세서”

## 1. 📌 문서 개요
### 1.1 목적
본 문서는 **전시(專時) – 전문가 유휴시간 파트타임/프리랜서 매칭 플랫폼**의
기능적·비기능적 요구사항을 정의하여, **기획·디자인·개발·운영**이 공통된 기준으로 제품을 설계·구현할 수 있도록 하는 것을 목적으로 한다.

### 1.2 범위
* 서비스 유형:
  * B2B/B2C 하이브리드 **전문가 시간 매칭 플랫폼**
* 주요 기능:
  * 전문가/기업 회원가입 및 프로필
  * 전문가 유휴시간(슬롯)·서비스 패키지 등록
  * 기업의 니즈/요청 등록 및 전문가 탐색
  * 시간 패키지 기반 매칭·계약·결제·정산
  * 리뷰/평판/분쟁처리, 운영자 백오피스
* 채널:
  * 웹(PC/모바일 웹) 1차 출시, 후순위로 앱 고려

### 1.3 주요 사용자(요약 페르소나)
* 전문가(공급 측)
  * 직장인 부캐 전문가 (개발, PM, 마케팅, 디자인 등)
  * 퇴직/경력단절 시니어 전문가 (재무, HR, 전략 등)
  * 풀타임 프리랜서/1인 기업
* 기업/고객(수요 측)
  * 초기 스타트업(1~30인)
  * 10~100인 성장기 스타트업/중소기업
  * 온라인 브랜드/커머스, 에이전시

### 1.4 용어 정의
* **전문가**: 특정 분야에서 유의미한 경력/실적을 가진 개인 또는 1인 기업
* **기업**: 전문가의 시간/서비스를 구매하는 법인 또는 개인 사업자
* **시간 패키지(Time Package)**:
  * `주 X시간`, `월 Y시간`, `월 N회 자문` 등 **시간 기반 상품**
* **슬롯(Slot)**: 전문가가 실제로 일할 수 있는 **요일·시간 단위 유휴시간**
* **매칭**: 기업의 요청과 전문가의 패키지/슬롯을 연결하는 과정

## 2. 🧩 시스템 개요

### 2.1 서비스 콘셉트
> “전문가의 남는 시간을 전시하고, 기업은 필요한 만큼만 빌려 쓴다.”
* 전문가: 유휴시간을 구조화하여 **부수입 + 포트폴리오** 확보
* 기업: 풀타임 채용 없이 **주/월 단위 파트타임 시니어** 활용

### 2.2 상위 기능 모듈
1. 사용자 관리/인증 모듈
2. 전문가 프로필·시간 패키지 모듈
3. 기업 요청·프로젝트 모듈
4. 매칭·추천 엔진
5. 계약·결제·정산 모듈
6. 리뷰·평판·분쟁 모듈
7. 알림/메시징(채팅) 모듈
8. 운영자 백오피스(Admin) 모듈

## 4. ⚙️ 비기능 요구사항 (NFR)
### 4.1 성능 (NFR-01)
| 요구사항        | 기준                                        |
| ----------- | ----------------------------------------- |
| 페이지 응답 시간   | 95% 요청에서 ≤ 1.5초                           |
| 검색/필터 결과 로딩 | ≤ 1초                                      |
| 동시 접속 사용자 수 | 최소 5,000명 (초기), 구조 상 10,000명까지 확장 가능하게 설계 |

### 4.2 보안 (NFR-02)
* TLS 1.2 이상(가능하면 1.3) HTTPS 통신 강제
* 비밀번호 해시(BCrypt/Scrypt 등) 및 비밀번호 정책(복잡도·만료 주기)
* 개인정보 암호화 저장(주민번호는 미수집, 계좌번호 등 민감정보 별도 암호화)
* 관리자/운영자 권한 분리 및 접근 로그 기록
* 정기적인 취약점 점검/모의 해킹(연 1회 이상 가이드)

### 4.3 가용성 & 신뢰성 (NFR-03)
* 월 가용성 목표 99.5% 이상
* 일 단위 자동 백업 + 주요 데이터는 최소 RPO 10분 이내(추후 조정)
* 장애 발생 시 RTO 30분 이내(중요 서비스 기준)

### 4.4 확장성 (NFR-04)
* 서비스/도메인 단위 모듈화(인증, 매칭, 결제, 메시징 등)
* Stateless API 서버 + 수평 확장 가능한 인프라 구성
* DB 샤딩/리플리카를 고려한 스키마 설계(읽기/쓰기 분리 가능 구조)

### 4.5 규제 및 법적 준수 (NFR-05)
* 개인정보보호법(PIPA) 및 정보통신망법 준수
* 전자상거래 관련 법규(결제/청약철회 등) 대응
* 전자세금계산서/현금영수증 발행 요건 대응(추가 설계 시 세무 자문 연계)

## 5. 🔌 시스템 인터페이스 요구사항 (간략)
* **외부 인증 API**: 소셜 로그인(OAuth2) – 네이버/카카오/구글
* **결제 PG**: 카드/계좌이체 지원 국내 PG사 연동 (토스페이먼츠/KG 이니시스 등)
* **이메일 발송 서비스**: SMTP 또는 외부 이메일 서비스 연동
* (선택) **타임트래킹/캘린더 연동**: 구글 캘린더 등 2차 기능으로 고려

## 6. 🧱 개발/배포 환경 (초안)
* FE: React or Next.js 기반 SPA/SSR, TypeScript
* BE: Node.js (NestJS 등) 또는 Spring Boot – 팀 역량에 따라 선택
* DB: PostgreSQL or MySQL 계열 RDB
* 인프라: AWS/GCP 등 클라우드, Docker/Kubernetes(초기에는 ECS/EC2도 가능)
* CI/CD: GitHub Actions / GitLab CI 등으로 자동 배포 파이프라인 구축

  * 기업 요청 등록 + 전문가 검색/추천
  * 기본 채팅, 계약/결제(단일 결제 플로우)
  * 간단한 리뷰 작성
  * 기본 운영자 페이지(사용자/거래 조회)

## 8. ✅ Acceptance Criteria (예시)
* [전문가] 직장인 부캐 PM이 **10분 이내**에 프로필·시간 패키지를 생성할 수 있다.
* [기업] 스타트업 대표가 요청 작성 후 **3분 이내**에 5명 이상의 후보 전문가 추천을 받는다.
* [거래] 월 100건의 거래가 발생해도 결제·정산·리뷰 플로우에 **에러율 1% 미만**.
* [운영] 운영자가 분쟁 신고를 접수하고, 관련 거래·채팅 이력을 **3클릭 이내**로 확인 가능.
